Публикация обезличенной demo API wiki в открытый GitHub-репозиторий

Рекомендуемый сценарий:

1. Создайте отдельный public-репозиторий, например:
   reqsy-demo-api-docs

2. В репозиторий можно залить:
   - либо только markdown-файлы как обычную документацию;
   - либо использовать GitHub Wiki, если репозиторий public.

3. Если используете обычный репозиторий без Wiki:

   - поместите файлы в корень репозитория или в каталог docs/
   - основную страницу можно сделать через README.md

4. Если используете GitHub Wiki:

   - включите Wiki у public-репозитория
   - создайте первую страницу
   - клонируйте wiki-репозиторий
   - скопируйте туда содержимое этой папки

5. Перед публикацией убедитесь, что:

   - нигде не осталось реальных доменов;
   - нигде не осталось реальных токенов;
   - нигде не раскрываются внутренние идентификаторы, схемы БД и коммерческие детали;
   - на главной странице явно указано, что это демонстрационная обезличенная версия.

6. Рекомендуемое описание репозитория:

   Sanitized demo API wiki for educational and documentation showcase purposes.
